# Q4：提出新的投票结合系统（Mechanism Design）

## 0. 我对题目的理解（聚焦 Q4）

题目要求提出一种比现有 Rank/Percent 更“公平”或在某种意义下“更好”（如更刺激、更能兼顾技术与参与感）的系统，并给出支持性证据。

这里的关键不是“提出一个花哨公式”，而是：

- 先**定义什么是更好**（指标体系）
- 再用 **Q1 后验样本驱动的仿真** 证明：新机制在这些指标上优于旧机制，并解释 trade-off。

## 1. 设计目标与约束（必须可执行、可解释）

### 1.1 设计目标（至少覆盖 4 个维度）

- **技术保护**：高评委分选手不应被断层人气轻易淘汰。
- **粉丝表达**：观众投票必须对结果有实质影响，否则节目失去互动。
- **鲁棒性**：对极端粉丝票分布（断层）不应过度敏感。
- **可解释/可执行**：规则简单，观众能理解，制作方能实施。

### 1.2 现实约束

- 仍需逐周淘汰，且必须能处理无淘汰/双淘汰周.
- 不依赖外生数据作为规则输入（外生数据最多用于我们离线评估与解释）。

## 2. 候选机制（给出可写进论文的公式）

记当周评委份额 `pJ_i` 与粉丝份额 `pF_i`（来自真实投票；在我们评估中由 Q1 后验样本提供）。

### 2.1 机制 A：粉丝票“压缩”后再相加（推荐主方案）

动机：Percent 机制在 `pF` 出现断层时，会把低评委高人气选手推到极端；压缩可缓和断层。

- 压缩函数（给 2 个可选版本）：
  - `g_1(p) = sqrt(p)`
  - `g_2(p) = log(p + eps)`
- 归一化：`pF'_i = g(pF_i) / sum_j g(pF_j)`
- 合并：`T_i = w * pJ_i + (1-w) * pF'_i`
- 淘汰：`argmin T_i`

参数：`w`（评委权重，0.5 为均衡）与 `eps`（数值稳定）。

### 2.2 机制 B：动态权重（前期互动，后期保技术）

动机：前期节目需要热度，后期更强调“实力选拔”。

- 设周权重 `w_t` 随周数递增：
  - `w_t = w_min + (w_max - w_min) * (t-1)/(T-1)`
- 合并：`T_{t,i} = w_t * pJ_{t,i} + (1-w_t) * pF_{t,i}`

### 2.3 机制 C：Percent + 常态化 Judge Save（制度化纠偏）

动机：S28 之后的“bottom-2 + judges decide”直觉上是在补技术保护。

- Step 1：按 Percent 得到 bottom-2
- Step 2：评委在 bottom-2 中淘汰评委分更低者（或带噪声版）

这套机制可作为“最容易被制作方接受”的推荐备选.

### 2.4 更多候选机制（用于对比与证明“最好”）

- 机制 D：对 `pF` 做 winsorize / cap（粉丝断层封顶后再归一化）。
- 机制 E：对 `pJ` 做非线性拉伸（例如 `pJ' ∝ pJ^gamma`），增强技术差异在后期的区分度.
- 机制 F：参数化机制族 `T_i(w, g) = w*pJ_i + (1-w)*g(pF_i)`，通过网格/优化搜索 Pareto 前沿（只在离线评估里做）。

## 3. 指标体系（我们如何证明“更好”）

对每个 season `s`、机制 `m`，通过仿真得到概率分布后，定义指标：

### 3.1 技术保护（Technical Protection）

- `TPI_s(m)`：冠军的评委排名分位数（与 Q2 一致，便于对照）。
- `LowJ_survival(m)`：评委分处于后 q 分位（如 bottom 30%）的选手进入决赛概率是否被过度放大.

### 3.2 粉丝表达（Fan Expression）

- `FanImpact(m)`：对 `pF` 施加小扰动时，冠军概率变化的平均幅度（灵敏度）。

### 3.3 鲁棒性（Robustness to fan outliers）

- 在极端情形（人为把一个选手 `pF` 放大到断层）下，机制导致“技术 top-1 被淘汰”的概率.

### 3.4 结果稳定性与可解释

- `Stability(m)`：机制下冠军概率分布的熵/集中度（过度随机也不好）。
- 规则复杂度：参数数量、解释成本（定性评分即可）。

## 4. 评估方法：用 Q1 后验做机制仿真（核心证据链）

### 4.1 输入

- 对每周，从 Q1 后验抽样 `pF_{s,t,*}`。
- 计算 `pJ_{s,t,*}`（由评委分归一化）。

### 4.2 仿真

- 对每个候选机制 `m`，重复模拟 M 次整季淘汰过程：
  - 记录冠军/前3 概率
  - 记录每周淘汰概率
  - 汇总指标（TPI/FanImpact/Robustness/Stability）

### 4.3 输出（写进报告的表与图）

- 表：每季各机制指标对比（含旧机制 baseline）
- 图：trade-off 曲线（例如 TPI vs FanImpact；Robustness vs FanImpact）
- 争议季案例：S27 Bobby Bones 作为重点展示新机制是否降低“低分高票夺冠”的概率，同时不让粉丝影响完全消失.

## 5. 推荐结论的写法（面向制作人的可落地建议）

### 5.1 推荐顺序（建议）

1. 首推 **机制 A（压缩粉丝票）**：改动小、解释直观、可调参.
2. 若制作方希望更强纠偏：结合 **机制 C（Judge Save）**.
3. 若希望赛季节奏更强：用 **机制 B（动态权重）**.

### 5.2 Memo 大纲（1–2 页）

- 背景：DWTS 历史争议与机制切换动因.
- 我们的发现：Rank vs Percent 的系统差异（引用 Q2 指标）。
- 新机制提案：一句话解释 + 公式 + 参数建议.
- 证据：关键指标对比表 + 争议季复盘图.
- 风险与执行：参数如何选、如何向观众解释、如何保留节目效果.

## 6. 风险与稳健性

- 新机制评估依赖 Q1 后验：必须报告“软约束 tau、先验强度 c、机制切换季”的敏感性.
- 如果某些季存在多淘汰/特殊规则：在仿真中按事实结构处理，否则会误判机制效果.
