# 2023 MCM Problem C（Wordle结果预测）完整题目与得分要点
## 一、完整题目内容
### （一）背景介绍
1. **Wordle游戏规则**  
Wordle是《纽约时报》每日推出的5字母单词猜谜游戏，玩家需在6次及以内猜出目标单词，每次猜测后获得颜色反馈：
- 绿色：字母在目标单词中且位置正确；
- 黄色：字母在目标单词中但位置错误；
- 灰色：字母不在目标单词中。  
游戏分普通模式与“困难模式”，困难模式要求玩家一旦找到正确字母（黄色/绿色tile），后续猜测必须包含这些字母。

2. **数据来源与范围**  
MCM提供2022年1月7日-2022年12月31日的每日游戏结果数据（来自Twitter用户分享），数据文件含以下核心字段：

| 字段 | 说明 |
|------|------|
| Date | 日期（mm-dd-yyyy格式） |
| Contest number | 游戏编号（202号起，对应2022年1月7日） |
| Word | 当日目标单词 |
| Number of reported results | Twitter上记录的总得分数量 |
| Number in hard mode | 困难模式玩家的得分数量 |
| 1-6 tries | 1-6次猜对的玩家百分比 |
| X（7 or more tries） | 6次未猜对的玩家百分比（因四舍五入，百分比可能不总和为100%） |


### （二）核心任务要求
需为《纽约时报》分析数据，回答以下5个关键问题，并提交不超过25页的PDF报告（含1页摘要、目录、完整解决方案、1-2页致编辑信件、参考文献）：
1. **报告结果数量的变异建模与预测**  
开发模型解释每日“报告结果数量”（Twitter分享得分总数）的波动，并用模型预测2023年3月1日报告结果数量的**预测区间**（需含置信范围，非单点预测）。

2. **单词属性与困难模式占比的关联分析**  
判断目标单词的属性（如字母频率、重复字母数、元音数量等）是否影响“困难模式玩家占比”，若有影响需量化说明，若无影响需解释原因。

3. **未来结果分布的预测模型**  
针对未来某日期的目标单词，建立模型预测结果分布（1-6次猜对及X的百分比），需：
- 分析模型不确定性来源；
- 以2023年3月1日目标单词“EERIE”为例给出具体预测；
- 评估模型置信度。

4. **单词难度分类模型**  
建立模型对目标单词进行难度分类（如简单/中等/困难），识别与难度相关的单词属性，用模型判断“EERIE”的难度，并讨论模型准确性。

5. **数据集其他有趣特征挖掘**  
探索数据中额外有价值的信息（如游戏流行度时间趋势、不同模式得分差异等）。


### （三）提交要求与限制
1. 仅可使用提供的Excel数据文件，无需访问《纽约时报》或Twitter官网，无额外外部数据可用；
2. 报告需包含摘要页、目录、完整解决方案、致编辑信件、参考文献，所有内容（含附录）总页数≤25页；
3. 需为报告中使用的想法、图像及其他材料注明来源。


## 二、得分要点（基于赛题要求与优秀论文经验）
### （一）数据预处理与特征工程（20%-25%）
1. **数据清洗**  
- 处理缺失值、异常值（如“报告结果数量”中的极端值），验证百分比字段的合理性（虽允许因四舍五入不总和为100%，但需排除明显错误数据）；
- 对日期字段进行时间特征提取（如工作日/周末、节假日、月份、季度，参考摘要6中“热点热度趋势”分析）。

2. **单词属性特征量化**  
- 基础属性：元音数量（如EERIE含3个E，元音占比60%）、重复字母数（EERIE含3个重复E）、字母频率（基于英文5字母单词库计算每个字母的出现概率，稀有字母占比高则难度可能更高）；
- 进阶属性：字母位置特征（如首字母是否为高频字母、关键位置（第2/3位）是否为稀有字母）、单词熟悉度（可通过英文常用词库匹配度量化，避免使用外部数据时可基于历史猜对率反向推断）；
- 特征有效性验证：通过相关性分析（如皮尔逊相关系数）筛选与“困难模式占比”“平均猜测次数”显著相关的特征（参考摘要1中“统计相关性分析”方法）。


### （二）核心模型构建（40%-45%）
#### 1. 报告结果数量预测模型（10%-12%）
- **模型选择**：优先使用时间序列模型（如ARIMA、LSTM，参考摘要4中“LSTM模型预测”、摘要6中“时间序列趋势分析”），需考虑时间趋势（如2022年游戏热度“先升后降”的热点特征）、季节性（周末分享量是否高于工作日）；
- **预测区间**：不可仅给出单点预测，需通过 Bootstrap 抽样或模型残差分析计算95%置信区间（体现不确定性）；
- **模型评估**：使用2022年后期数据（如10-12月）进行交叉验证，通过MAE、RMSE、MAPE等指标评估预测误差（参考摘要4中“MSE、RMSE和R²评估”）。

#### 2. 单词属性与困难模式占比关联模型（8%-10%）
- **分析方法**：若为线性关联，可使用多元线性回归（分析各单词属性对“困难模式占比”的回归系数）；若为非线性关联，可使用决策树、随机森林（参考摘要1中“xgboost、lightGBM”）；
- **结论支撑**：需通过显著性检验（如P值）判断属性是否有影响，例如“重复字母数增加是否显著提高困难模式占比”，若无关需结合玩家行为逻辑解释（如选择困难模式仅与个人偏好相关，与单词无关）。

#### 3. 结果分布预测模型（12%-15%）
- **模型设计**：因需预测7个百分比（1-6次+X），需采用多输出预测模型（如多输出LSTM、多变量线性回归，参考摘要4中LSTM应用）；
- **不确定性分析**：需明确不确定性来源（如数据随机性（Twitter样本代表性）、特征未覆盖的变量（如当日突发热点影响分享意愿）、模型参数波动），可通过敏感性分析（调整关键参数观察预测结果变化，参考摘要4中“COV≈0.4的敏感性分析”）量化不确定性；
- **EERIE预测示例**：需结合EERIE的单词属性（高重复元音、字母频率），给出具体百分比（如参考摘要4中优秀论文结果：2%, 11%, 25%, 24%, 19%, 14%, 5%），并说明预测逻辑（如重复元音导致位置判断难度增加，X比例略高于平均值）。

#### 4. 单词难度分类模型（10%-12%）
- **难度定义**：需先明确量化标准（如以“平均猜测次数≥5次”或“X比例≥15%”定义为“困难”，避免主观分类）；
- **模型选择**：分类算法（如逻辑回归、随机森林、SVM，参考摘要1中“logit回归、xgboost”），需进行交叉验证（如5折交叉验证）；
- **准确性评估**：通过混淆矩阵、准确率、召回率、F1分数评估模型（参考摘要1中“ROC曲线、混淆矩阵图评估”），例如“困难单词”的召回率需≥80%，避免漏分；
- **EERIE难度判断**：结合模型输出与单词属性，若EERIE的“平均猜测次数”预测值≥4.5次、X比例≥12%，可归类为“中等偏难”，需解释关键影响因素（如重复E导致黄色反馈误导，增加试错次数）。


### （三）数据分析与特征挖掘（15%-20%）
- **基础趋势分析**：2022年游戏流行度变化（如1-3月报告结果数量上升，6月后下降，验证“热点热度衰减”规律，参考摘要6）；
- **模式差异分析**：困难模式与普通模式的得分分布对比（如困难模式下X比例是否显著高于普通模式，1-2次猜对比例是否更低）；
- **字母特征关联**：高频字母（如E、A、T）是否对应更低的平均猜测次数，稀有字母（如Q、Z）是否导致X比例上升；
- **异常日期分析**：识别报告结果数量或X比例异常的日期，结合时间特征（如节假日）或单词属性解释原因（如某节日当天分享量激增，或某稀有单词导致X比例翻倍）。


### （四）报告完整性与学术规范（15%-20%）
1. **结构完整性**  
- 摘要页：需浓缩核心结论（如EERIE难度等级、2023年3月1日报告结果数量预测区间），控制在1页内；
- 致编辑信件：以简洁语言向《纽约时报》谜题编辑汇报关键发现（如“重复字母单词难度更高，建议平衡每日单词难度分布”），避免技术术语，控制在1-2页；
- 参考文献：规范引用赛题给定的4个参考来源，若使用统计方法需引用方法出处（如ARIMA模型的原始文献）。

2. **逻辑与可视化**  
- 模型逻辑：每个模型需说明“为什么选该模型”（如LSTM适合捕捉时间序列的长期依赖，适合报告结果数量预测）、“假设条件”（如假设2023年无重大事件影响游戏热度）；
- 可视化呈现：使用折线图展示时间序列趋势、热力图展示单词属性与难度的相关性、箱线图展示不同难度单词的得分分布（参考摘要1中“初步可视化分析”要求）。

3. **不确定性与局限性讨论**  
- 明确模型局限性（如未考虑2023年《纽约时报》可能调整游戏规则、Twitter用户样本无法代表所有玩家）；
- 提出改进方向（如未来可加入玩家 demographics 数据优化预测，或结合游戏内交互数据（如首次猜测单词）提升难度分类准确性）。


### （五）创新性（5%-10%）
- **方法创新**：如将“熵权法+层次分析法”结合（参考摘要1）量化单词难度，或使用注意力机制优化LSTM模型（提升结果分布预测的准确性）；
- **视角创新**：如从“玩家心理”角度解释困难模式选择（如高熟悉度单词可能吸引更多玩家尝试困难模式），或结合“热点衰减模型”（参考摘要6）预测长期报告结果数量趋势。
