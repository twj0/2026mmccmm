## 快速上手

### 环境配置
项目使用 `uv` 进行依赖管理和环境隔离。

- **同步环境：** `uv sync`(默认)
- **安装所有依赖（含机器学习/深度学习）：** `uv sync --all-groups`

### 执行流程
整个分析流程可以通过统一入口一键复现：

- **运行主线流程：** `uv run python run_all.py`
- **运行主线 + 扩展展示（Showcase）：** `uv run python run_all.py --showcase`

### 质量控制
- **代码规范检查：** `uv run ruff check .`
- **自动化测试：** `uv run pytest`

## 设计理念

本框架围绕**可复现性**、**模块化**与**自动化**核心原则设计。

### 1. 数据完整性 (Data Integrity)
- **原始数据层 (`data/raw/`)：** 存放不可变的初始输入文件。脚本禁止修改此目录下的任何内容。
- **处理数据层 (`data/processed/`)：** 存储经过清洗和标准化后的统一数据集，确保所有后续分析步骤均基于一致的数据源。

### 2. 模块化逻辑 (`src/`)
分析代码采用软件工程化的组织方式：
- *核心配置 (`core`)* 统一管理项目路径和全局变量，消除硬编码。
- *数据处理 (`data`)* 封装 I/O 操作与审计逻辑，保障数据流转的安全。
- ***分析流水线 (`pipelines`)*** 将不同的逻辑模块拆分为独立的脚本，便于独立开发、调试和复用。

### 3. 自动化产出 (`outputs/`)
所有结论（图表、表格、预测结果）均由程序自动生成。
- `outputs/` 目录作为流水线的“草稿板”，任何最终报告中引用的证据必须能通过脚本复现。

### 4. 文档驱动 (Documentation-First)
在 `docs/` 目录中维护详细的设计规格说明和分析计划，确保分析逻辑的透明度与可维护性。

## 目录结构说明

- `src/`: 核心业务逻辑与流水线定义。
- `data/`: 多级数据存储（原始 -> 处理后）。
- `outputs/`: 自动生成的图表、表格与数据产物。
- `docs/`: 技术规格说明、任务定义与执行计划。
- `paper/`: 最终报告的草稿与模板。
- `tests/`: 稳定性测试与逻辑校验脚本。
